資料安全架構
🔐
SMART on FHIR OAuth 2.0
病人可透過 SMART on FHIR 授權機制查看個人資料，Access Token 15 分鐘短效。牙醫師以執照身份即可調閱病人跨院紀錄，無需病人每次個別授權。
🔒
TLS 1.3 全程加密
所有 FHIR API 傳輸均經 TLS 1.3 加密，EV SSL 憑證每 90 天自動更新。
🛡
Patient ID 與身分資料分離
系統僅流通 PID，身分證號加密儲存於獨立資料庫。
🔍
AuditEvent 稽核
每筆資料讀寫均產生 FHIR AuditEvent，保存 7 年。
📋
Consent resource 授權管理
病人可透過 FHIR Consent 細粒度管理哪間診所可讀、有效期限設定。
🏥
跨院資料互通
每間診所有唯一 ORG ID，持有合法執照的牙醫師可直接查詢病人在任何診所的歷史 Observation，不需病人額外授權，實現真正的跨院資料共享。
💾
localStorage 本地持久化（Demo）
Demo 環境使用 localStorage 持久化，關閉瀏覽器資料仍保留。
